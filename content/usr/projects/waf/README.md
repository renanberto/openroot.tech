# AWS WAF Security Cases

Credential stuffing mitigation and edge protection patterns.

## Problem

Credential stuffing attacks often arrive through noisy, repetitive traffic patterns.
The goal is to slow down abusive behavior without damaging legitimate users.

## What it covers

- AWS WAF rules
- CloudFront protection
- Bot Control and CAPTCHA strategy
- user-agent and request behavior filtering
- operational visibility through logs and metrics
- security tradeoffs for real-world applications

## Stack

- AWS WAF
- CloudFront
- Bot Control
- CAPTCHA
- CloudWatch

## What it demonstrates

This project shows practical security engineering: understanding traffic, choosing controls, measuring impact and protecting accounts at the edge.
